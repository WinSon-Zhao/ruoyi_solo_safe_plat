# 阶段0：基础框架与运维验证平台

> **版本**：V1.0  
> **日期**：2026年5月  
> **目标**：基于 ruoyi-solon 改造，搭建安全平台基础框架，实现一体化运维验证平台  
> **原则**：复用 ruoyi-solon 已有能力，减少重复开发，页面操作替代多工具切换

---

## 1. 项目分析

### 1.1 ruoyi-solon 现状分析

```
ruoyi-solon
├── ruoyi-admin          # 接口模块 [8080] - 单体入口
├── ruoyi-common         # 通用工具 - 可复用
├── ruoyi-framework      # 框架核心 - 安全/拦截器/AOP等
├── ruoyi-gen            # 代码生成 - 可选
├── ruoyi-schedule       # 定时任务 - 可复用
├── ruoyi-solon-ui       # Vue3前端 - 可复用或改造
├── ruoyi-system         # 系统管理 - 核心复用模块
└── sql/ry_solon.sql     # 初始化脚本
```

**ruoyi-solon 已具备能力**：
- ✅ 用户/角色/菜单/部门/岗位管理
- ✅ 字典管理（动态配置）
- ✅ 参数管理
- ✅ 操作日志/登录日志
- ✅ 定时任务调度
- ✅ 服务监控（CPU/内存/磁盘）
- ✅ 缓存监控
- ✅ 代码生成
- ✅ Sa-Token 认证（比 JWT 更完善）
- ✅ 数据权限控制
- ✅ MyBatis-Flex 集成

### 1.2 改造策略

| ruoyi-solon 模块 | 处理方式 | 说明 |
|------------------|----------|------|
| ruoyi-common | 保留复用 | 工具类、常量、注解等 |
| ruoyi-framework | 保留复用 | 安全配置、拦截器、过滤器 |
| ruoyi-system | 保留复用 | 用户/角色/菜单/字典等核心表 |
| ruoyi-schedule | 保留复用 | 定时任务能力 |
| ruoyi-admin | 改造扩展 | 作为 security-platform-admin |
| ruoyi-solon-ui | 改造扩展 | 增加运维验证页面 |
| ruoyi-gen | 可选保留 | 用于生成业务代码 |

**新增模块**：
- `security-platform-api` - **数据总线**（模块间服务接口定义，单体→类调用，微服务→RPC）
- `security-platform-common` - 安全平台通用（扩展 ruoyi-common）
- `security-platform-config` - 动态配置、脚本引擎
- `security-platform-device` - 设备管理
- `security-platform-parse` - 日志解析
- `security-platform-analysis` - 安全分析
- `security-platform-alert` - 告警管理
- `security-platform-disposal` - 运维处置
- `security-platform-report` - 数据上报
- `security-platform-ops` - **运维验证平台（阶段0核心）**

> **架构演进说明**：阶段0-3 基于 ruoyi-solon 单体版开发，所有模块打包为一个 JAR。阶段4 迁移到 ruoyi-cloud-solon 微服务版时，`api` 模块自动切换为 RPC 调用，核心业务代码零修改。详见 [00-调研与总体设计文档.md §9.3](./00-调研与总体设计文档.md)。

## 2. 阶段0 目标与范围

### 2.1 核心目标

1. **基础框架搭建**：基于 ruoyi-solon 改造，保留其系统管理能力
2. **运维验证平台**：一体化页面，完成所有基础设施的操作验证
3. **开发环境就绪**：开发人员只需启动项目，即可在页面完成所有验证和运维操作
